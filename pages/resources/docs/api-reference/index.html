Introduction

Welcome to the AI Cryptopia API documentation. Our API is organized around REST principles, uses JSON for request/response payloads, and relies on standard HTTP response codes.

Base URL

https://api.aicryptopia.io/v1

Note: This API documentation outlines our planned endpoints. The API is currently under development and will be implemented in future releases.

Authentication

AI Cryptopia uses JWT tokens for API authentication. You'll need to include the token in the Authorization header: 

Authorization: Bearer YOUR_JWT_TOKEN
POST /auth/login

Authenticate user and receive JWT token

{
  "email": "user@example.com",
  "password": "secure_password"
}

Response

{
  "token": "eyJhbGciOiJIUzI1NiIs...",
  "user": {
    "id": "user_123",
    "email": "user@example.com"
  }
}

AI Services

POST /ai/analyze

Analyze data using AI models

{
  "data": "Data to analyze",
  "model": "sentiment_analysis",
  "parameters": {
    "confidence_threshold": 0.8
  }
}
POST /ai/train

Submit training data for AI models

{
  "model_id": "model_123",
  "training_data": [...],
  "parameters": {
    "epochs": 10,
    "batch_size": 32
  }
}

Blockchain

GET /blockchain/transactions/{address}

Get transaction history for an address

Response

{
  "address": "0x...",
  "transactions": [
    {
      "hash": "0x...",
      "value": "1.5 ETH",
      "timestamp": "2024-01-17T..."
    }
  ]
}
POST /blockchain/smart-contracts/deploy

Deploy a new smart contract

{
  "contract_name": "AIModel",
  "parameters": {
    "initial_state": {...},
    "gas_limit": 3000000
  }
}

User Management

GET /users/{user_id}

Get user profile information

PUT /users/{user_id}

Update user profile

{
  "name": "John Doe",
  "preferences": {
    "notification_enabled": true
  }
}

Data Services

POST /data/ipfs/upload

Upload data to IPFS

{
  "data": "Base64 encoded data",
  "metadata": {
    "name": "example.json",
    "type": "application/json"
  }
}
GET /data/ipfs/{cid}

Retrieve data from IPFS by CID
